# 특전 설계 의논점

## 1. 타이밍 충돌 — p1 특전 vs G1/G6 대응 불가

### 문제
- **C6 핀포인트 블라인드** (`phase: p1`): 상대가 배율을 못 보고 커밋하게 만드는 효과
- **G1 봉쇄** / **G6 특전복사**: 현재 `setTarget`이 EYE_OPEN 페이즈에서만 호출 가능
- C6를 봉쇄하거나 복사하려면 타겟의 커밋 **전**에 선언해야 하지만 — contract 상 불가능

### 영향받는 특전 조합
| 공격 특전 | 타겟 특전 | 문제 |
|---|---|---|
| G1 봉쇄 | C6 핀포인트 블라인드 | 봉쇄 선언이 EYE_OPEN → 상대는 이미 커밋 완료, 봉쇄 무의미 |
| G6 특전복사 | C6 핀포인트 블라인드 | 복사 후 쓸 타이밍(커밋 전) 없음 |

### 잠재적 충돌 확장
p1 효과를 가진 특전이 추가될수록 G1/G6의 대응 범위가 좁아짐.

### 선택지
1. **C6를 p1에서 제거** — 커밋 후 정보를 숨기는 방향으로 재설계 (가장 간단)
2. **G1/G6에 pre-commit 타겟 선언 추가** — `setPreTarget(roundId, target)`을 OPEN 페이즈에 허용
3. **C6 효과 재정의** — "LOCKED 이후 상대의 리빌 정보를 가린다"처럼 타이밍 변경

### 결정
- **C8 순서 교란**: 삭제 예정
- **C6 핀포인트 블라인드**: 삭제 예정
- **G1 봉쇄 / G6 특전복사 / G7 픽 미러링**: `setTarget` 제거, 자신을 제외한 랜덤 1명에게 자동 적용으로 변경
- **G2 처형**: 타겟 지정 제거 → 눈치게임에서 순서가 겹친 상대에게 자동 발동. 3명 전원 겹치면 나머지 2명 중 랜덤 1명 처형

## 2. C4 탈락 은폐 — 존재 가치 의문

### 효과
- `getSurvivingCount` (C2 대응): 생존 픽 수 조회 차단
- `getOneSurvivingPick` (C3 대응): 최고배율 생존 픽 조회 차단
- `getOverlappingNibbles` (C1 대응): 겹침 계산에서 본인 제외

### 문제점

**1. 능동적 효과가 없는 순수 카운터 특전**
C1/C2/C3 보유자가 없으면 완전히 무의미. 상대 특전을 커밋 시점에 모르는 상태에서 C4를 선택하는 건 도박에 가까움.

**2. 온체인 데이터라 완전한 은폐가 불가능**
reveal 후에는 `commitments[roundId][player].pickedMask`가 온체인에 공개됨. view 함수만 차단하는 것이라 스토리지 직접 조회나 이벤트 로그로 우회 가능 — UI 레벨의 차단에 불과.

### 비교
| 특전 | 차단 범위 | 독립 가치 |
|---|---|---|
| G3 저지불가 | 상대 모든 특전 효과 | 있음 |
| C4 탈락 은폐 | C1/C2/C3 view 함수만 | 없음 (카운터 전용) |

### 선택지
1. **삭제** — C1/C2/C3 자체가 강하지 않아서 카운터 특전까지 필요한지 의문
2. **효과 강화** — 단순 차단을 넘어 독립적인 가치 부여 (예: 본인 픽 정보를 SETTLED까지 완전 비공개)
3. **유지** — C1/C2/C3가 강해질수록 자연히 가치가 올라올 수 있음

## 3. B3 라스트 스탠드 — 카테고리 재분류

### 현재
- 카테고리: B (눈치게임형)

### 문제

### 결정

## 4. F 카테고리 함정형 — 페널티 미설계

### 현재 상태
- F1 숫자 함정: 해당 배율 절반 감소
- F2 구간 함정: 해당 구간 픽 전부 제거
- F3 순서 함정: 포기 픽 +1
- F4 이중 함정: F1+F2 동시 적용

함정이 **발동되지 않았을 때** (상대가 함정을 피했을 때) 함정 설치자에게 돌아오는 패널티가 없음. 함정 설치 자체가 무비용이라 전략적 긴장감 부족.

### 결정
- 추후 설계 예정 (예: 불발 시 자신에게 역페널티, 함정 정보 공개 등)

### 밸런스 이슈

#### E 카테고리 전반 — 보너스 수치가 낮음
조건이 까다롭지만 보상이 작음. 조건을 맞추는 픽 제약이 생기는데 이득이 +0.15~+0.3pt 수준.
- E5 구간 분산: +0.15pt — 조건 대비 보상 의문
- E1 서로소 보너스: 서로소 4픽 조건이 빡빡한데 +0.2
- E4 레인 선언: 확률 낮은 조건에 +1.0pt만으로 충분한지

#### G 카테고리 랜덤화 — str 특전 약화
타겟 지정 제거 후 랜덤 적용으로 변경 시:
- G1 봉쇄: 원하는 상대를 못 잡을 수 있음 → 실전 가치 하락
- G6 특전 복사: 약한 특전 보유자에게 걸리면 역효과
- G4 강제 교환: 저배율 보유자에게 걸리면 손해
강도 재조정이 필요할 수 있음.

#### B3 라스트 스탠드 D 이동 후 B 카테고리 — str 공백
B에 str이 없어짐. 눈치게임 특전 중 고위험-고수익 선택지 부재.

#### C 카테고리 — 삭제 후 정보 특전 밀도 저하
C5/C6/C8 삭제 시 C1/C2/C3/C7/C9만 남음. 정보 획득(C1~C3)과 전체 교란(C7/C9) 사이의 중간 단계 없음.

#### wk 특전 과잉 — A 카테고리
A2/A3/A4/A5가 모두 wk~mid 수준. 조건이 상충되는 경우(A1과 A2는 겹침 수 조건이 반대)가 있어 공존 가능하지만, A 카테고리 안에서 차별화 부족.

### 전략 다양성 점검 (추가 논의 예정)

기본 픽 배율과 눈치게임 순서 배율만 따라가는 "정석 루트"가 지배적인지, 아니면 특전과 배율 분배가 아래 루트들도 실제로 경쟁력 있게 만드는지 검증 필요.

- **정석 루트**: 고배율 픽 집중 + 눈치게임 1번/2번 선택
- **역발상 루트**: 3등 순서 의도 선택, 저배율 픽 집중 (E3 공백 선점, B2 고독한 질주 등 활용)
- **도박 루트**: 극단적 조건 특전으로 정석 루트를 이기는 경우가 이론상 가능한지

확인 항목:
1. 최고배율 픽 4개 + 1번 순서 조합의 기대 점수
2. 저배율 픽 + 역발상 특전 조합의 최대 기대 점수
3. 두 루트 간 기대값 차이가 특전 없이도 벌어지는지
4. 특전이 차이를 좁히는지 혹은 오히려 정석 루트를 더 강화하는지

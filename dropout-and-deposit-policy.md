# Dropout And Deposit Policy

현재 게임은 commit-reveal 기반이고, 일부 단계는 keeper가 자동 진행을 보조한다.
이 구조에서는 단순 트롤링을 위해 게임을 흔드는 여지가 예전보다 줄어들기 때문에, 금전적 보증금을 반드시 둘 필요성은 크지 않다.

특히 실제 판돈 시스템을 도입하지 않을 예정이라면, 보증금을 통한 경제적 억제보다 규칙 기반 불이익 처리 쪽이 더 적절하다.

## 보증금이 꼭 필요하지 않은 이유

- commit 이후 플레이어의 선택이 사실상 고정됨
- keeper가 일부 진행을 자동화해 단계 정지를 줄여줌
- 무보상 게임이라 경제적 악용 유인이 낮음
- 금전 패널티는 UX와 진입 장벽을 높일 수 있음

따라서 현재 방향에서는 보증금보다 `자동 불이익 규칙`으로 대응하는 편이 더 자연스럽다.

## 중간 이탈 처리 원칙

핵심 원칙은 다음과 같다.

- 돈으로 벌주지 않는다
- 행동을 하지 않으면 자동으로 불리한 결과를 받는다
- 라운드는 가능한 한 멈추지 않고 끝까지 진행한다

## 단계별 처리 제안

### 1. pick reveal 미제출

현재는 keeper 구조이므로, 사용자가 필요한 정보를 정상 전달했다면 keeper가 자동으로 처리한다.
반대로, reveal에 필요한 정보를 끝내 제출하지 않으면 다음과 같이 처리한다.

- `revealed = false`
- 해당 라운드 점수 0
- 생존 픽 없음으로 간주

효과:
- reveal 미참여자가 게임 결과를 흐리지 않음
- 다른 플레이어의 진행을 막지 않음

### 2. eyeCommit 미제출

눈치 순서를 정하지 않으면 자동으로 가장 불리한 기본값을 적용한다.

예:
- 자동으로 `3`번 순서 부여
- 또는 eye 배수 보너스 없음 처리

효과:
- 순서 선택을 하지 않아도 게임은 계속 진행됨
- 의도적 미제출이 큰 이득으로 이어지지 않음

### 3. eyeReveal 미제출

현재 구조에서는 keeper가 대행할 수 있으므로, 실제 문제는 사용자가 끝까지 `order + salt`를 전달하지 않은 경우다.

이 경우:
- `eyeRevealed = false`
- eye 관련 보너스 없음
- overlap 판정에서도 불리하게 처리 가능

효과:
- 마지막 단계 이탈이 라운드 정산을 막지 않음
- 미전달 플레이어만 손해를 보게 됨

### 4. 완전 이탈자

라운드 도중 아무것도 더 하지 않는 플레이어는 다음처럼 처리할 수 있다.

- 해당 라운드 점수 0
- 결과 화면에 이탈 표시
- 누적 이탈률 또는 매너 점수에 반영

효과:
- 경제적 패널티 없이도 반복 이탈 억제 가능
- 사회적/게임 내 평판 차원에서 정리 가능

## 추천 방향

무보상 게임 기준으로는 다음이 가장 적절하다.

1. 보증금은 두지 않는다
2. 각 phase 미수행 시 자동 불이익을 준다
3. 라운드는 중단 없이 끝까지 자동 진행되게 만든다
4. 반복 이탈은 점수/평판/표시로 관리한다

## 결론

현재 게임 구조에서는 보증금 없이도 운영이 가능하다.
중요한 것은 금전적 억제가 아니라, `행동을 하지 않으면 자동으로 손해를 보게 하는 규칙`을 명확히 두는 것이다.

즉 이 프로젝트에서는 보증금보다 다음 두 가지가 더 중요하다.

- 라운드가 멈추지 않도록 자동 진행을 보장하는 것
- 중간 이탈자에게는 점수/보너스 측면의 자동 불이익을 주는 것
